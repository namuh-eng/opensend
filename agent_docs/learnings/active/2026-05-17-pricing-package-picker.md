## Pricing tiers should render as package choices, not a continuous slider

OpenSend Cloud tiers are discrete pooled API + broadcast packages with non-linear quota jumps. Avoid a Resend-like continuous rail/slider because equal visual spacing between 5k, 55k, 100k, 120k, 250k, 500k, and Custom implies a smooth numeric scale that the product does not sell. Use a segmented package picker that labels both the plan family and included volume, preserving the combined-plan strategy while making the UI distinct from Resend.
