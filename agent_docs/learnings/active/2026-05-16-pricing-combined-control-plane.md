## Combined cloud pricing is a deliberate product strategy

Public and authenticated billing surfaces should present one OpenSend Cloud plan family covering API sends, broadcast fanout, contacts, segments/topics, domains, webhooks, and usage visibility. Do not show Resend-style separate Transactional vs Marketing checkout tabs until the app has separate Stripe SKUs plus marketing-specific contact, broadcast, automation-run, and quota enforcement.

Current plan copy should say that one plan covers API sends and broadcasts rather than labeling the picker as only transactional email. The 2026-05-16 public limits are Free 5k, Starter 55k with 10 domains, and Growth 120k with 1,000 domains.
